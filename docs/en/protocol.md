# Protocol Layer

The shared protocol contract between verifier ↔ attester ↔ relying-party, defined in `protos/attestation.proto`, compiled to Rust code at build time via `tonic-build`, and exposed by the `protos` crate. This crate contains no business logic and only carries gRPC services/messages.

## Flow Overview (RATS background-check)

```
RP --GetEvidence(tee_type, nonce)--> attester --AA/TEE--> evidence
RP <-(evidence, wasm_component)-- attester
RP --Verify(tee_type, nonce, evidence, wasm_component)--> verifier
RP <-(EAR JWT)-- verifier
RP locally verifies EAR signature + compares eat_nonce == local nonce
```

The nonce is generated by the RP itself (32 random bytes recommended) and does not depend on the verifier issuing a challenge token. The replay window is governed by the RP holding the association between nonce and EAR; the verifier is stateless.

## gRPC Services

| Service | Method | Caller | Description |
|---|---|---|---|
| `AttesterService` | `GetEvidence` | RP → attester | Push nonce, receive evidence |
| `VerifierService` | `Verify` | RP → verifier | Submit evidence, receive EAR |

See `protos/attestation.proto` for full message field definitions.

`VerifyRequest.wasm` is a `oneof` with two options:
- First submission: `wasm_component` (wasm byte stream)
- Subsequent reuse: `wasm_component_id` (stable ID returned by verifier after first submission)

## TeeType Enum

| proto value | kebab name (used in claims) | Notes |
|---|---|---|
| `MOCK = 1` | `mock` | Skip real verification |
| `CCA = 2` | `cca` | ARM CCA |
| `CCA_HYDRA = 3` | `cca-hydra` | CCA + hydra zk |
| `TDX = 4` | `tdx` | Intel TDX |
| `TDX_HYDRA = 5` | `tdx-hydra` | TDX + hydra zk |
| `CSV = 6` | `csv` | Hygon CSV |
| `CSV_HYDRA = 7` | `csv-hydra` | CSV + hydra zk |
| `ITRUSTEE = 8` | `itrustee` | iTrustee |
| `VIRTCCA = 9` | `virtcca` | VirtCCA |

The attester config `tee_type` is a kebab string; the `tee_type` in the request uses proto enum values. The RP client's `parse_tee_type` translates between the two. The attester rejects requests with a `tee_type` different from its own configuration.

## Nonce Encoding

| Usage | Encoding |
|---|---|
| `GetEvidenceRequest.nonce` / `VerifyRequest.nonce` | raw bytes (proto `bytes`) |
| RP-side logging / EAR `eat_nonce` | base64url no-pad string |
| CCA evidence JSON `nonce` field | base64url no-pad |
| AA REST `runtime_data` parameter | standard base64 |
| hydra `challenge` public input | `nonce_to_scalar(raw_bytes)` |

The attester and wasm appraiser must encode/decode using the same rules, or nonce comparison will fail.

## EAR Output

The EAR issued by the verifier is an ES256 JWT. Top-level claims:

```text
iss            = "unified-attestation-verifier"
iat            = unix seconds (issue time)
exp            = unix seconds (expiry, iat + 3600)
eat_profile    = "tag:github.com,2024:unified-attestation"
eat_nonce      = base64url(RP nonce)
tee_type       = "mock" | "cca" | "cca-hydra" | "csv" | "csv-hydra" | "tdx" | "tdx-hydra" | "itrustee" | "virtcca"
component_id   = wasm component ID
verifier_id    = { developer: "unified-attestation" }
submods        = claims map returned by wasm (including per-TEE measurements, see below)
trust_vector   = { instance_identity, configuration, executables } (dynamically assigned, see below)
```

The RP can verify the signature locally, decode, and compare `eat_nonce == local nonce` using the verifier's public key:

```bash
relying-party \
    --attester http://127.0.0.1:9000 \
    --verifier http://127.0.0.1:8080 \
    --tee-type mock \
    --pubkey config/keys/ear_public.pem
```

`executables < 2` is considered untrusted.

### Per-TEE Claims (in submods)

CCA path (`cca` / `cca-hydra`):

| Field | Source | Description |
|------|------|------|
| `cca_realm_initial_measurement` | injected after host verification | Realm Initial Measurement (hex), core trust measurement |
| `cca_realm_personalization_value` | injected after host verification | Realm personalization value (hex) |
| `cca_platform_instance_id` | injected after host verification | CCA platform instance ID (hex) |
| `cca_platform_implementation_id` | injected after host verification | CCA platform implementation ID (hex) |
| `cca_platform_lifecycle` | injected after host verification | Platform security lifecycle: `secured` / `secured_no_debug` / `recoverable` / `not_secured` |
| `cca_platform_sw_components` | injected after host verification | Platform software component list |
| `nonce_bound` | wasm appraiser check | Whether nonce binding succeeded |
| `roots_hex` | hydra appraiser check | (cca-hydra only) shrubs root list |
| `subject` | hydra appraiser extract | (cca-hydra only) device identity, from `cca_platform_instance_id` |

CSV path (`csv` / `csv-hydra`):

| Field | Source | Description |
|------|------|------|
| `chip_id` | injected after host verification | Chip serial number |
| `measurement` | injected after host verification | Measurement value (hex) |
| `vm_version` | injected after host verification | VM firmware version (hex) |
| `policy_nodbg` | injected after host verification | Policy: debug disabled (0/1) |
| `policy_noks` | injected after host verification | Policy: key sharing disabled (0/1) |
| `nonce_bound` | wasm appraiser check | Whether nonce binding succeeded |
| `roots_hex` | hydra appraiser check | (csv-hydra only) shrubs root list |
| `subject` | hydra appraiser extract | (csv-hydra only) device identity, from `chip_id` |

TDX path (`tdx` / `tdx-hydra`):

| Field | Source | Description |
|------|------|------|
| `mr_td` | wasm appraiser extract | TD measurement (hex) |
| `mr_seam` | wasm appraiser extract | SEAM module measurement (hex) |
| `mr_config_id` | wasm appraiser extract | Config ID (hex) |
| `report_data` | wasm appraiser extract | Report data binding value (hex) |
| `tcb_status` | wasm appraiser extract | TCB status: UpToDate / SWHardeningNeeded / OutOfDate / ... |
| `advisory_ids` | wasm appraiser extract | Applicable security advisory IDs |
| `nonce_bound` | wasm appraiser check | Whether nonce binding succeeded |
| `roots_hex` | hydra appraiser check | (tdx-hydra only) shrubs root list |

iTrustee path (`itrustee`):

| Field | Source | Description |
|------|------|------|
| `nonce_bound` | wasm appraiser check | Whether nonce binding succeeded |
| `uuid` | injected after host parsing | Trusted Application UUID |
| `ta_img` | injected after host parsing | TA image measurement (hex), optional |
| `ta_mem` | injected after host parsing | TA memory measurement (hex), optional |
| `hash_alg` | injected after host parsing | Hash algorithm, optional |
| `version` | injected after host parsing | TA version, optional |
| `ima_log_size` | wasm appraiser extract | IMA log size (bytes), optional |

Evidence collection: attester fetches `{report, ima_log}` via AA REST and attaches the nonce. Host verification: parses report JSON to extract TA measurements and injects them into evidence. Full verification requires libteeverifier.so (deploy when available).

VirtCCA path (`virtcca`):

| Field | Source | Description |
|------|------|------|
| `nonce_bound` | wasm appraiser check | Whether nonce binding succeeded |
| `token_size` | injected after host parsing | CBOR/COSE token size (bytes) |
| `cert_size` | injected after host parsing | Device certificate size (bytes) |
| `ima_log_size` | wasm appraiser extract | IMA log size (bytes), optional |
| `event_log_size` | wasm appraiser extract | Event log size (bytes), optional |

Evidence collection: attester fetches `{evidence, dev_cert, event_log}` via AA REST and attaches the nonce. Host verification: full verification requires OpenSSL + cose + ciborium (CBOR/COSE decoding + HW cert chain verification). Deploy when available.

### Trust Vector Dynamic Assignment

`trust_vector` is no longer hardcoded; it is dynamically set based on verification results:

| TEE Type | `instance_identity` | `configuration` | `executables` |
|---------|---------------------|-----------------|---------------|
| **mock** | 2 | 2 | 2 |
| **CCA / CCA-Hydra** | nonce_bound ? 2 : 0 | secured/secured_no_debug→2, not_secured/recoverable→1, unknown→0 | 2 |
| **CSV / CSV-Hydra** | nonce_bound ? 2 : 0 | 2 | 2 |
| **iTrustee / VirtCCA** | nonce_bound ? 2 : 0 | 2 | 2 |
| **TDX / TDX-Hydra** | 2 | 2 | UpToDate→2, SWHardeningNeeded/ConfigurationAndSWHardeningNeeded→1, OutOfDate/Revoked→0, unknown→1 |

AR4SI values: 2 = Affirming, 1 = Warning, 0 = None.
