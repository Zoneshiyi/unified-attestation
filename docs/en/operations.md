# Operations Manual

Command reference for building, running, and debugging.

## Prerequisites

- Rust 1.90.0 (see `rust-toolchain.toml`)
- `cargo install cargo-component --locked` (for building wasm appraisers)
- `rustup target add wasm32-wasip1`
- `openssl` (for generating ES256 key pairs)

## Common Scripts

| Script | Purpose | Dependency |
| ----------------------------- | ---------------------------- | ------------------------------ |
| `scripts/run-mvp.sh` | mock mode end-to-end (no TEE dependency) | — |
| `scripts/gen-keys.sh` | Generate ES256 key pair into `config/keys/` | openssl |
| `scripts/build-appraisers.sh` | Build all wasm appraisers | cargo-component, wasm32-wasip1 |

`config/keys/` and `config/hydra-shrubs/` are generated by scripts and gitignored.

End-to-end test steps for each TEE require the corresponding hardware environment. Command references in per-TEE docs:

- CCA / CCA + hydra: [cca.md](cca.md)
- Hygon CSV / CSV + hydra: [csv.md](csv.md)
- TDX / TDX + hydra: [tdx.md](tdx.md)
- iTrustee: requires iTrustee TEE hardware + libqca.so + libteeverifier.so (test commands below)
- VirtCCA: requires VirtCCA TEE hardware + libvccaattestation.so + OpenSSL (test commands below)

### iTrustee End-to-End Test

```bash
bash scripts/gen-keys.sh
bash scripts/build-appraisers.sh
cargo build --release -p verifier -p attester -p relying-party

ttrpc-aa &
api-server-rest --features attestation &

./target/release/verifier --config config/verifier-itrustee.toml > /tmp/verifier-itrustee.log 2>&1 &
./target/release/attester --config config/attester-itrustee.toml > /tmp/attester-itrustee.log 2>&1 &
sleep 2

./target/release/relying-party \
    --attester http://127.0.0.1:9000 \
    --verifier http://127.0.0.1:8080 \
    --tee-type itrustee \
    --pubkey config/keys/ear_public.pem \
    --ear-out /tmp/ear-itrustee.jwt
```

## Tools

| Command | Purpose |
| -------------------------------------------------------------------------- | --------------------------- |
| `cargo run -p hydra --bin setup_keys -- <root_count> <path_len> [out_dir]` | Trusted setup, generate PK/VK |
| `cargo run -p hydra --example shrubs_roots` | Compute trusted root hex from device list |
| `sha256sum target/wasm32-wasip1/release/*.wasm` | Compute appraiser sha256 (for whitelist updates) |
